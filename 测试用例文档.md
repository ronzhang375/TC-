| 项目 | 要求 |

| MySQL | 8.0 |
| Redis | 7.0 |
| 后端端口 | 8080 |
| 小程序端口 | 微信开发者工具 |

### 1.2 启动步骤

```bash
# 1. 启动 MySQL + Redis（Docker 方式）
cd lottery-mall-dev/docker
docker-compose up -d

# 2. 初始化数据库
mysql -u root -p < ../sql/init.sql

# 3. 启动后端
cd ..
mvn spring-boot:run

# 4. 启动小程序（用 HBuilderX 打开 lottery-mall-app）

# 5. 启动管理后台
cd lottery-mall-admin
npm install
npm run dev
```

## 附录：发现的问题详情

### BUG-001 OrderService CartItemVO 引用错误
- **位置**：`module/order/service/OrderService.java`
- **现象**：代码中 `List<CartItemVO> cartItems = cartService.getList(userId)` 引用了 `CartItemVO`，但内部定义的内部类与 `cart` 模块的 VO 字段不一致
- **修复**：引入 `cart` 模块的 `CartItemVO`，删除内部冗余类
- **状态**：✅ 已修复

### BUG-002 缺少 CommissionDetailMapper
- **位置**：`module/commission/service/CommissionSettleService.java`
- **现象**：Service 引用 `commissionDetailMapper`，但未定义对应的 Mapper 接口
- **修复**：创建 `CommissionDetail` 实体类和 `CommissionDetailMapper` 接口
- **状态**：✅ 已修复

### BUG-003 ChannelBillController 类引用混乱
- **位置**：`module/commission/controller/ChannelBillController.java`
- **现象**：导入了多个未使用的类（ServiceImpl、Page 等），且 ChannelCommissionSummary 类位置错误
- **修复**：清理 imports，删除冗余导入
- **状态**：✅ 已修复
